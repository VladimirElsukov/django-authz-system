# django-authz-system
Django-проект, связанный с системой аутентификации и авторизации пользователей.Проект представляет собой серверную часть веб-приложения, 
предназначенную для реализации собственного механизма аутентификации и авторизации. Главная цель — предложить настраиваемое решение 
разграничения прав доступа для сложных корпоративных решений, управляемых вручную через административную панель.

Основные возможности:
Регистрация и авторизация пользователей
Регистрация пользователей с указанием ФИО, адреса электронной почты и пароля.
Авторизация пользователей по адресу электронной почты и паролю.
Разграничение прав доступа
Создание произвольных ролей и присвоение конкретных разрешений (например, чтение, написание, обновление записей).
Управление ролевыми полномочиями через административный интерфейс.
Пользовательские профили
Возможность редактирования персональных данных зарегистрированных пользователей.
Безопасное удаление аккаунтов («мягкое удаление») без физического стирания данных из базы.

Архитектура и схемы данных
1. Модель пользователя (User) пример: Полностью наследует функциональность стандартной модели пользователя Django с добавлением полей для полного имени пользователя и признака активности:
python 
class User(AbstractUser):
    first_name = models.CharField(max_length=150)
    last_name = models.CharField(max_length=150)
    middle_name = models.CharField(max_length=150, blank=True)
    email = models.EmailField(unique=True)
    password = models.CharField(max_length=128)
    is_active = models.BooleanField(default=True)

2. Роли и права доступа
Используется классическая двухуровневая модель:
Роль (Role) — категория пользователя (например, администратор, редактор, читатель).
Правило (Permission) — конкретные действия, доступные пользователю в рамках конкретной роли (например, просмотр, создание, удаление).
Модель роли:
python

class Role(models.Model):
    name = models.CharField(max_length=100)
    
Модель правила(разрешение):
python
class Permission(models.Model):
    role = models.ForeignKey(Role, on_delete=models.CASCADE)
    resource = models.CharField(max_length=100)
    action = models.CharField(max_length=100)
    
3. Контроль доступа к данным
При каждом обращении пользователя к ресурсу производится проверка наличия соответствующего разрешения:
Если разрешение найдено, возвращается ресурс.
Если пользователь не авторизован, выдаётся ошибка HTTP 401 Unauthorized.
Если пользователь авторизован, но требуемого ресурса у него нет, выводится сообщение об ошибке HTTP 403 Forbidden.


Установка и запуск проекта
1. Требования к окружению
Python версии >= 3.8
PostgreSQL / SQLite (для локального тестирования) На этапе разработки выбираем SQLite( так как работа предстоит на разных ПК)
Docker (нет)
2. Инструкция по установке
   
Клонируйте репозиторий:
bash
git clone https://github.com/yourusername/django-authz-system.git
cd django-authz-system

Установите зависимости:
bash
pip install -r requirements.txt

Создайте конфигурационный файл .env с параметрами подключения к базе данных и секретными ключами:
bash
cp .env.example .env

Выполните миграции базы данных:
bash
python manage.py migrate

Запустите сервер разработки:
bash
python manage.py runserver

Тестирование
Проект поставляется с набором тестовых данных, которые позволяют продемонстрировать работу функциональности:

Пользователи: admin@*******.com/password, editor@********.com/password
Правила доступа установлены для ресурсов "Users" и "Articles".
Чтобы протестировать функциональность, выполните POST-запросы на адрес регистрации и авторизации через Swagger-интерфейс или curl-команды.
